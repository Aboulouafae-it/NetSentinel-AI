# GitHub Visual Assets

This document defines the visual asset rules for NetSentinel AI on GitHub.

## Social Preview

Recommended dimensions:

- 1280 x 640 px
- PNG or JPG
- Dark NetSentinel AI visual identity
- Clear text: `NetSentinel AI`
- Supporting text: `Network Operations • Cybersecurity • Wireless Diagnostics`

Rules:

- Use original artwork only.
- Do not use Debian/Kali/Ubuntu/vendor logos as NetSentinel product identity.
- Do not include copyrighted network/security artwork.
- Do not include customer topology, IP addresses, screenshots with secrets, or
  unredacted logs.
- Do not use unreviewed AI-generated commercial artwork without owner approval.

## README Logo

If a README logo is added later:

- Prefer SVG owned by the project.
- Keep it readable on GitHub light/dark backgrounds.
- Do not imply vendor endorsement.
- Store source assets under `docs/assets/` or a reviewed branding directory.

## Screenshot Aspect Ratios

Recommended:

- App screenshots: 16:9, at least 1440 x 810.
- Desktop screenshots: 16:9, at least 1920 x 1080.
- Boot screenshots: full VM viewport, crop only if no important context is lost.

## Allowed Asset Sources

- Real captures from the running NetSentinel AI app.
- Real captures from NetSentinel AI OS VM sessions.
- Original project SVG/PNG assets.
- Diagrams created from Mermaid or project-owned vector sources.

## Disallowed Asset Sources

- Vendor logos used as product identity.
- Customer screenshots or topology diagrams.
- Raw packet capture visualizations with sensitive data.
- Screenshots containing tokens, secrets, private keys, credentials, or
  unredacted cloud account details.
