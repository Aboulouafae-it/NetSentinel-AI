# Code Of Conduct

Be respectful, constructive, and security-conscious.

NetSentinel AI deals with network operations, cybersecurity monitoring, device
telemetry, logs, and infrastructure diagnostics. Contributors and users must not
pressure anyone to publish private infrastructure details, customer data,
secrets, exploit instructions, or raw captures.

## Expected Behavior

- Communicate respectfully.
- Keep technical feedback specific and actionable.
- Protect sensitive operational data.
- Report vulnerabilities privately through `SECURITY.md`.
- Be honest about MVP, roadmap, and production-readiness limits.

## Unacceptable Behavior

- Harassment or personal attacks.
- Posting secrets, customer data, raw captures, private keys, or credentials.
- Publishing exploit details in public issues.
- Misrepresenting roadmap features as production-ready.
- Using vendor/distribution branding in a way that implies endorsement.

Maintainers may edit, hide, or remove public content that exposes sensitive data
or creates security risk.
